# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Running the app

There is no build step, package manager, or test suite. The site is a static export from QGIS's Qgis2threejs plugin (v2.8) plus two custom scripts. Open `index.html` directly in a browser to run it (see `Simple Instructions.txt`). If the browser blocks `file://` from loading `data/index/scene.js`, serve the folder statically (e.g. `python3 -m http.server`) from the repo root.

Browser-side debugging only: there are no linters, formatters, tests, or CI.

## Architecture

The 3D scene represents Point Lobos / Monterey Bay bathymetry. The runtime is assembled in `index.html` in this order:

1. **Vendored libraries (do not edit):** `threejs/three.min.js`, `threejs/OrbitControls.js`, `threejs/DeviceOrientationControls.js`, `threejs/ViewHelper.js`, `proj4js/proj4.js`, `fontawesome/*`.
2. **`Qgis2threejs.js` + `Qgis2threejs.css`:** the viewer framework generated by the QGIS plugin. Exposes the global `Q3D` with `Q3D.application` (camera, renderer, scene, controls, `measure` tool, `cameraAction`) and `Q3D.gui` (popup, layer panel). Treat this as third-party; upstream is https://github.com/minorua/Qgis2threejs.
3. **`data/index/scene.js`:** the baked scene (~100 MB). Loaded via `app.loadSceneFile`. It calls `app.loadJSONObject({...})` with `properties.proj` set to `+proj=utm +zone=10 +datum=WGS84` — the scene's native CRS is UTM 10N, which is why waypoint labels need proj4 to convert back to WGS84 lon/lat. Do not hand-edit this file; regenerate it from QGIS.
4. **`mobile.js`:** Qgis2threejs' mobile/AR template. Defines `Q3D.Config.AR`, overrides `app.start/pause/resume`, swaps `OrbitControls` ↔ `DeviceOrientationControls` when AR mode toggles, handles geolocation (`getCurrentPosition`, `moveToCurrentLocation`, `zoomToCurrentLocation`), and wires the on-screen buttons (AR toggle, crosshair, layers, settings, info). `init()` is called from `index.html` after `app.init(container)`.
5. **`custom.js` + `custom.css`:** our dive-specific enhancements. Loaded last, via the `waitForQ3D` polling loop at the bottom of `index.html`, so it can safely monkey-patch `app.measure`.

### How `custom.js` extends the measure tool

`custom.js` wraps three methods on `Q3D.application.measure` — `addPoint`, `removeLastPoint`, `clear` — to keep two sets of HTML overlay labels in sync with the measure tool's `markerGroup` children:

- **Waypoint labels** (one per marker): `lat, lon<br>depth ft`. Position is converted 3D → map coords via `scene.toMapCoordinates`, then map XY → WGS84 lon/lat via `toLonLatXY()` (which prefers `proj4` with the scene's CRS, falls back to Web Mercator, then to treating coords as already-degrees). Anchored top-right of the projected marker, toggled globally with **Shift+L** (persisted in `localStorage` key `q3d_waypointLabelsVisible`).
- **Segment labels** (one per edge between consecutive markers): distance in feet + magnetic heading. `MAG_DEC = -12.0` is Monterey Bay's declination; the sign convention in the code is `magnetic = true + MAG_DEC` (East declination stored as negative). If this map is ever used elsewhere, update `MAG_DEC` in `custom.js`.

Both label sets are plain `<div class="label">` elements in `document.body`; their screen positions are recomputed every frame by a `requestAnimationFrame` projector loop that calls `vec3.project(app.camera)`. The styling lives in `custom.css` (the `.label` rules and the canvas `z-index: 0` override that keeps labels on top).

`custom.js` also recolors Qgis2threejs' measure line to `0xffff66` (`LINE_BRIGHTNESS`) for underwater visibility, and runs a one-shot `centerCameraOnSceneWhenReady()` that frames the camera on the scene's bounding box after the first visible mesh appears.

### Things to watch out for

- `Qgis2threejs.js` and `data/index/scene.js` are generated artifacts. Fixes belong in `custom.js` / `custom.css` / `mobile.js` / `index.html`, not in the generated files.
- The `waitForQ3D` loader in `index.html` is load-order-sensitive: `custom.js` assumes `app.measure` exists when it patches it. If you add new scripts, keep them after that loader or gate them the same way.
- AR mode depends on `navigator.mediaDevices.getUserMedia` and `DeviceOrientationControls`, which require HTTPS (or `localhost`) and user permission; they will silently fail on plain `http://` served over LAN.
- Plain `W` is bound by Qgis2threejs to wireframe mode and `Ctrl+W` closes the browser tab — that's why the waypoint-label toggle is Shift+L. Pick similarly non-conflicting keys for any new hotkeys.
- The `Pt Lobos 3D V3.zip` at the repo root is a distribution bundle, not source.
